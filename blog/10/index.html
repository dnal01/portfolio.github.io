GIT Version control commands

Global set up

  • git config --global user.nameSets the default name for Git to use when you commit
  • git config --global user.email-//-
  • git config --global init.defaultBranch Sets the default branch to the name "main" instead of "master"
  • git config --global core.editor Sets the default editor to VSC
  • git config --list

Creating an ssh key for your computer

  • ssh-keygen Hit enter twice. This generates two files, one ending in .pub. This is the public key that you can share with others.
  • cat ~/.ssh/id_rsa.pub displays public key. Paste it into GitHub account settings > SSH keys > add new. You'll be able to clone/push to GitHub from this computer for authorized repositories!

Adding SSH key to the GitHub

  • In the gitHub go to the Settings
  • Select SSH and GPG keys
  • Click New SSH key

Synchronizing local repository with remote GitHub

Before setting up Git you need to initialize a repository
  • git remote add origin GitHubRepositoryURL Synchronize Git with desktop
  • git remote -v to view configured remotes
  • git push -u origin main first time

    -u flag is used to set origin as the upstream remote in your git config. You only have to do that the first time and then can use push and pull without arguments.

  • git push after that. Use this after git add and git commit to push changes to the GitHub.
  • git pull
  • git pull origin main Before you start working on a repo that day.
  • git fetch The way you update the .git folder to contain new information from the remote. But keeps your local branches intact yet.

Restoring changes

  • git restore fileName restores files in working repository (not added or committed).
  • git reset Unstage a file and keep changes unless you restore it
  • git restore --staged fileName Alternative for a reset.
  • git reset --soft HEAD~1 Uncommiting, but keeps changes
  • git reset --hard HEAD~1 Uncommiting, but removing (unstaging) changes
    • HEAD The most recent commit is called like that. You can also use hash number of a specific commit.
    • ~1 Move back by one commit
    • --soft keep changes
    • --hard undo changes
  • git revert --no-commit HEAD~1 Inverts the changes made by the commit and generates a new commit with the inverted content. Used to save history.

Branches

  • git branch See the current branch
  • git switch -c branchName create a branch
    • switch Hops me over to that branch.
    • -c Creates new branch.
  • git merge anotherBranchName -m "" merge branchName with a current branch where you are
  • git push -u origin branchName - for the first push for branch
  • git branch -d branchName Deletes a local branch.
  • git push -d origin branchName - removes remote branch

General

  • git pull --rebase Replaces local lifecycle from the moment when deviation occurred, with remote one. Plus, adding new commits on the top one by one merging all the local commits, starting from the first deviated, first with the last remote commit, then with the each last merge.
  • git commit --amend This will replace the last commit with a new one. It's best to do this only with commits that haven't been shared yet. If you need to fix your last commit, maybe to change the message or add something you forgot.

Logs and reports

  • git diff branch1..branch2 Compares HEADS of two branches.
  • git log --all --oneline --graph
    • -all Shows a log throughout all branches
    • --oneline Makes each commit in one line
    • --graph Highlights commits of different branches
