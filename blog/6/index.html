  2. Combining selectors

    • p.class Class selector. All p elements with this class;
    • .class1.class2 only the elements where those two classes are applied at the same time;
    • section > p Child selector. Only on a level below, i.e. direct child;
    • section p Descendant selector. On any lower level. And it is not limited to element selectors, e.g.: .color > p;
    • a, p Selectors in CSS can be separated by comma, to indicate more then one selector for rules

    The Box Model

    • box-sizing: border-box; Whatever width you specify it will absorb borders and paddings as well, so it will be the total width from border to border, including the content. And it is a good practice to stick to this option rather then to content-box.
    • margin-right; margin-left; cumulates their margins if left or right boxes have any too. but margin-top; margin-button; do not they are collapsed and the bigger margin wins. But it is not true for floated elements.
    • overflow

    Relative and Absolute

    • Static Position

      Being in a static position - normal document flow, i.e. not Relative and Absolute, the element will ignore any offsets (right, left, top...).

    • Relative

      An element is positioned relative to its normal position, i.e. its original spot is preserved. Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

    • Absolute

      An element with position: absolute; is positioned relative to the nearest positioned ancestor (positioned relative or absolute).

      However; if an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.

      Note: Absolute positioned elements are removed from the normal flow, and can overlap elements.

    Inheritance

    Some elements are not Inherited, meaning they do not give their property to child elements, as e.g. box-sizing.

    This behavior can be overwritten by adding such elements to a * {} property, which literally during rendition adds properties to each of the element.

    Short hands without "-" overwrites above properties with "-".

  7. Responsive website - Logical operators

    • AND. @media (min-width: 768px) and (max-width: 991px) {...}.
    • OR. @media (min-width: 768px), (max-width: 991px) {...}.
    Do not overlap range boundaries! It is best practice to include media after declaring of other styles.

    Devise sizes

    • Large - (min-width: 1200px) {...}
    • Medium - (min-width: 992px) and (max-width: 1199px) {...}

    Sizing

    meta name="viewport" content="width=device-width, initial-scale=1.0" This will set the viewport of your page, which will give the browser instructions on how to control the page's dimensions and scaling.

    Use the "vw" unit when sizing the text. 10vw will set the size to 10% of the viewport width.

    style="font-size:5vw;"

    Viewport is the browser window size. 1vw = 1% of viewport width. If the viewport is 50cm wide, 1vw is 0.5cm.
