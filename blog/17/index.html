    • Working with lists

    It is indexed by range of numbers

    • varName[-1] access a list from the end
    • varName[3:5] access a list in diapason by the rule [ start - inclusive : end - exclusive]
    • varName[:5] access a list in diapason starting from index 0
    • varName[3:] access a list in diapason ending wit the last index (including)
    • varName[3:] = ['one', 1, 'two', 2] replaces list wth a new values in a diapason
    • varName + ['newElement', 'newElement'] adds a new elements into a list

      • Cooping lists

    • newVar = list(existingVar) or newVar = existingVar[:] creates a new list in memory, because if using "=", the new variable will point to the same address in memory where the existed list was.

      Note any other operators, where you do not simply assign variables, but perform some operations, will also create a new list, e.g. newVar = existingVar + [newEl1, newEl2]

        Pandas` methods

        • newVarTable.index = ['firstIndexName', 'secondIndexName'...] renames default 0, 1, 2... indexes onto specified names

          • Accessing pandas row an column

        • newVarTable.loc["index-name1"] Label-based, returns a value with specified row.
        • newVarTable.loc[["index-name1", "index-name2"...]] Label-based, returns table with entire rows with the specified index.
        • newVarTable.loc[["index-name1", "index-name2"...], ["column-name1, column-name2"...]] returns specific indexes (rows) with specific columns
        • newVarTable.loc[:, ["column-name1, column-name2"...]] returns all rows with specific columns
        • newVarTable.iloc[[1,2,3], [0, 1]] Integer position-based, returns 3 rows and 2 columns
